# 페이지 및 라우트 명세

청약닷컴의 페이지별 상세 명세서입니다. 각 페이지의 목적, 데이터 요구사항, 레이아웃, 인터랙션, 수용 기준을 정의합니다.

---

## 공통 요소

### 하단 내비게이션 바 (모바일)

- **위치:** 하단 고정, 글래스모피즘 표면 (`color-bg-page` @ 80% + `backdrop-blur: 20px`)
- **항목:** 홈 (`/`), 청약 (`/listings`), 뉴스 (`/news`), 검색 (오버레이)
- **활성 상태:** `brand-primary-500` 아이콘 + 라벨, 비활성은 `neutral-400` 사용
- **높이:** 64px + 기기 하단 인셋 (안전 영역)

### 상단 헤더 (데스크톱, ≥1024px)

- **위치:** 상단 고정, 글래스모피즘 표면
- **레이아웃:** 로고 (좌측) | 내비게이션 링크 (중앙) | 검색 아이콘 (우측)
- **내비게이션 링크:** 홈, 청약, 뉴스
- **로고:** 프로젝트 루트의 `logo.svg` 사용

### 통합 검색 (오버레이)

- **실행:** 네비게이션 검색 아이콘 클릭 또는 `⌘K` (`Ctrl+K`) 단축키
- **범위:** 청약 (이름, 위치, 시공사) 및 뉴스 (제목, 본문) 전체 검색
- **UI:** 전체 화면 모달 오버레이 (배경 딤), 중앙 패널 (최대 640px)
- **결과:** 유형별 그룹화 (청약, 뉴스), 그룹당 최대 5개 미리보기
- **최근 검색어:** localStorage에 저장, 최대 10개
- **키보드:** `Escape`로 닫기, 열릴 때 입력란 자동 포커스

### 푸터

- **스타일:** 최소한의 디자인, `surface` 배경
- **링크:** 소개, 이용약관, 개인정보처리방침
- **내용:** 저작권 표시, 로고 마크

## 페이지 1: 홈 대시보드

- **라우트:** `/`
- **목적:** 주간 청약 일정과 시장 인사이트를 한눈에 보여주는 화면. "이번 주 청약은 뭐가 있지?"라는 질문에 답하는 첫 화면.

### 섹션

#### 1.1 히어로 / 주요 청약

- 가장 주목할 만한 예정 청약을 강조하는 전체 너비 카드
- 내용: 아파트명, 위치, 주요 일정, 세대수, 시공사명
- 배경: `white` 카드 위에 은은한 그라데이션 오버레이
- CTA: `/listings/[id]`로 연결되는 "상세 보기" 버튼

#### 1.2 이번 주 일정 타임라인

- 가로 스와이프 카드 캐러셀 (모바일) / 그리드 행 (데스크톱)
- 카드별 내용: 아파트명, 위치 요약, 접수 기간, 상태 칩
- 상태 칩: 접수중은 `success`, 마감임박은 `warning`, 접수예정은 `info` 사용
- 접수 시작일 기준 오름차순 정렬
- 빈 상태: "이번 주 예정된 청약이 없습니다" + 전체 목록 링크

#### 1.3 시장 인사이트 카드

- 2열 그리드 (모바일) / 3열 그리드 (데스크톱)
- 카드별 내용: 지표명, 수치, 추세 표시 (증감 화살표 + 백분율)
- 지표: 평균 경쟁률, 이번 달 총 세대수, 인기 지역
- 배경: `surface` 페이지 배경 위 `white` 카드

#### 1.4 전국 실거래가 TOP 5

- 최근 2일 거래 중 거래금액 상위 5건을 순위형 행 리스트로 노출
- 행별 내용: 순위(1~3위 브랜드 강조 / 4~5위 톤다운), 아파트명, 지역(시군구+동), 공급면적이 아닌 전용면적 + 평 병기(`85㎡ (34평)`), 층, 거래일, 거래금액(우측 강조)
- 섹션 부제: "최근 2일 거래 · 거래금액 상위 · 출처: 국토교통부"
- 배열이 비었거나 상위 요청이 실패하면 섹션 전체를 숨김
- 상세 페이지 링크 없음 (실거래 상세/검색 페이지 신설 후 후속 티켓에서 연결)
- 면적 축 주의: 청약 리스팅은 공급면적(`minSupplyArea`/`maxSupplyArea`) 기준, 실거래는 전용면적(`exclusiveArea`) 기준 — 서로 다른 축이므로 라벨 혼동 방지

#### 1.5 최신 뉴스 미리보기

- 최근 뉴스 기사 3개를 간결한 카드로 표시
- 카드별 내용: 카테고리 칩, 제목 (2줄 제한), 날짜
- `/news`로 이동하는 "전체 보기" 링크

### 데이터 요구사항

| 엔드포인트 | 데이터 |
|---|---|
| `GET /main/featured` | 주요 청약 단일 객체 (공급면적 기준) |
| `GET /main/weekly-schedule` | 월~금 일자별 공고 목록 |
| `GET /main/stats` | 이번 달 경쟁률·공급세대 통계 + 인기 지역 |
| `GET /main/top-trades` | 거래금액 상위 최근 실거래 5건 (전용면적 기준) |
| `GET /api/news/latest?limit=3` | 최신 뉴스 기사 3개 |

### 모바일 레이아웃

- 세로 스택: 히어로 → 일정 캐러셀 → 실거래 TOP 5 → 뉴스 미리보기
- 일정 섹션은 스냅 포인트가 있는 가로 스크롤 사용
- 아래로 당겨 새로고침 시 전체 데이터 리로드

### 주요 인터랙션

- 청약 카드 탭 → `/listings/[id]`로 이동
- 뉴스 카드 탭 → `/news/[id]`로 이동
- 일정 캐러셀 가로 스와이프
- 아래로 당겨 전체 대시보드 데이터 새로고침

### 수용 기준

- [ ] 히어로 섹션에 주요 청약의 모든 필수 필드가 렌더링됨
- [ ] 주간 일정에 이번 주 해당 청약이 올바르게 표시됨
- [ ] 상태 칩이 청약 상태별로 올바른 색상 토큰을 사용함
- [ ] 시장 인사이트 카드에 추세 방향과 백분율이 표시됨
- [ ] 실거래가 TOP 섹션에 순위 1~5위가 거래금액·지역·전용면적(평 병기)·층·거래일과 함께 노출되고, 데이터가 비었을 때 섹션이 숨겨짐
- [ ] 4G 모바일 환경에서 2초 이내 로드 (LCP < 2s)

## 페이지 2: 청약 목록

- **라우트:** `/listings`
- **목적:** 필터링 및 검색이 가능한 전체 아파트 청약 목록. 사용자가 청약 기회를 탐색하는 주요 화면.

### 섹션

#### 2.1 고정 필터 바

- **위치:** 헤더 아래 고정 (데스크톱) 또는 상태 바 아래 고정 (모바일)
- **스타일:** `white` 배경에 은은한 하단 그림자
- **필터 (Phase 6 기준 `/apt-sales` 에 바인딩):**

| 필터 | 유형 | 옵션 | 쿼리 키 |
|---|---|---|---|
| 키워드 (단지명) | 텍스트 + 지우기 버튼, 300ms 디바운스, ≤50자 | 자유 문자열 부분 일치 | `q` |
| 지역 (시/도) | 다중선택 드롭다운, 수도권/광역시/도 그룹 | 전국 17개 시/도 | `region` |
| 상태 | 다중선택 칩 | 접수예정, 접수중, 발표대기, 발표일, 청약완료 | `status` |
| 청약 유형 | 다중선택 칩 | 공공 (public), 민간 (private) | `type` |

- 다중선택 칩은 활성 시 `Check` 아이콘을 앞에 붙입니다 (색상 전용 금지 규칙)
- 모바일 필터 아이콘에 활성 필터 개수 배지 표시
- "초기화" 버튼은 모든 필터(상태/유형/지역/키워드)를 한 번에 비웁니다

**로드맵 (백엔드 미지원으로 현재 미구현):**

| 필터 | 차단 사유 |
|---|---|
| 구/군 | `regionCode` 가 시/도 단위까지만 제공 |
| 공급 유형 (특별/일반) | 쿼리 파라미터 없음. 공급 분해는 상세 엔드포인트에서만 노출 |
| 시공사 | 쿼리 파라미터 없음. `constructorName` 은 아이템 응답에만 포함 |

#### 2.2 정렬 컨트롤

- **옵션:** 날짜순 (최신순, 기본값), 날짜순 (오래된순), 인기순 (조회수)
- **스타일:** 결과 위 우측 정렬 인라인 드롭다운

#### 2.3 청약 카드

- **레이아웃:** 전체 너비 카드 스택 (모바일) / 2열 그리드 (데스크톱)
- **카드 내용:**
  - 아파트명 (헤드라인)
  - 위치 (시/도 + 구/군 + 동)
  - 일정: 접수 기간 (시작일–종료일)
  - 상태 칩 (접수중 / 접수예정 / 마감)
  - 시공사명
  - 세대수 및 면적 범위 (예: "84㎡ ~ 114㎡")
- **카드 스타일:** `white` 배경, `8px` 반경, 테두리 없음 (디자인 시스템 준수)
- **인터랙션 상태:** 호버/프레스 시 배경이 `surface`로 전환

#### 2.4 페이지네이션

- SEO를 위한 페이지 기반 내비게이션 (무한 스크롤 아님)
- 표시: 이전 / 페이지 번호 / 다음
- URL에 현재 페이지 반영: `/listings?page=2`
- 페이지당 20개 항목

### 데이터 요구사항

| 엔드포인트 | 데이터 |
|---|---|
| `GET /apt-sales` | 페이지네이션된 청약 목록 (orval 생성 `useGetAptSalesList` 로 바인딩) |
| **쿼리 파라미터 (바인딩 완료)** | `status[]`, `houseDetailType[]`, `regionCode[]`, `keyword`, `page`, `size` |
| **쿼리 파라미터 (로드맵)** | `district` (구/군), `supply` (공급유형), `builder` — 백엔드 미지원. API 도착 전까지 UI 노출 보류 |

지역 옵션은 orval 이 생성한 `ItemRegionCode` enum 에서 가져오며 별도
`/filters/*` 엔드포인트를 호출하지 않습니다. 구 문서에 남아있던
`/api/filters/regions`, `/api/filters/builders` 표기는 무효입니다.

### 모바일 레이아웃

- 필터 바가 필터 아이콘 + 활성 필터 개수 배지가 있는 단일 행으로 축소
- 필터 아이콘 탭 시 전체 화면 필터 시트 (하단 드로어) 열림
- 카드는 전체 너비, 16px 세로 간격으로 분리
- 스크롤 중에도 고정 필터 바 유지

### 주요 인터랙션

- 필터 선택 → 결과 즉시 갱신 (별도 제출 버튼 없음)
- 카드 탭 → `/listings/[id]`로 이동
- 페이지 변경 → 상단으로 스크롤, URL 갱신
- 필터 상태가 URL 쿼리 파라미터에 유지되어 공유 가능

### 수용 기준

- [ ] 6가지 필터 유형이 각각 및 조합으로 동작함
- [ ] 지역 변경 시 구/군 드롭다운이 갱신됨
- [ ] URL에 모든 활성 필터와 현재 페이지가 반영됨
- [ ] 카드에 모든 필수 필드가 올바른 상태 칩 색상과 함께 표시됨
- [ ] 필터 조건에 맞는 청약이 없을 때 빈 상태가 표시됨

## 페이지 3: 청약 상세

- **라우트:** `/listings/[id]`
- **목적:** 하나의 청약에 대한 완전한 정보 제공. 사용자가 자격 요건과 다음 단계를 판단하는 의사결정 페이지.

### 섹션

#### 3.1 단지 개요 헤더

- 아파트명 (디스플레이 헤딩)
- 지도 핀 아이콘이 있는 위치 정보
- 시공사명 및 시공사 사이트 링크 (외부)
- 주요 통계 행: 총 세대수, 면적 범위, 예상 분양가 범위
- 상태 칩 (대형)
- 작성자 또는 편집자 정보 표시 금지

#### 3.2 일정 타임라인

- 모든 단계를 보여주는 세로 타임라인:
  1. 모집공고
  2. 특별공급 접수
  3. 일반공급 1순위
  4. 일반공급 2순위
  5. 당첨자 발표
  6. 계약 기간
  7. 입주 예정
- 각 단계에 날짜와 현재/과거/미래 상태 표시
- 현재 단계는 `primary` 강조색으로 하이라이트
- 지난 단계는 `text-low`, 향후 단계는 `text-mid` 사용

#### 3.3 공급 내역 표

- **특별공급:**
  - 유형: 기관추천, 다자녀, 신혼부부, 생애최초, 노부모부양
  - 유형별: 세대수, 면적 옵션, 해당 시 소득/자산 기준
- **일반공급:**
  - 순위 단계: 1순위, 2순위
  - 선정 방식 요약 (가점제 vs 추첨제)
  - 순위별: 면적별 세대수
- 표 스타일: 테두리 없음, 교대 행에 `surface` 배경 사용

#### 3.4 자격 요건 간편 확인 (선택적 기능)

- 접을 수 있는 섹션에 주요 자격 기준 표시
- 소득 기준, 자산 기준, 거주 요건
- 계산기가 아닌 참고 정보 제공 목적

#### 3.5 관련 뉴스

- 해당 청약 또는 해당 지역 관련 뉴스 기사 최대 5개
- 간결한 카드 형식: 제목, 날짜, 카테고리 칩
- `/news/[id]`의 전체 기사로 링크

#### 3.6 공식 링크

- 청약홈 (applyhome.co.kr) 바로가기
- 시공사 공식 사이트
- 모집공고 PDF 다운로드 (있는 경우)
- 가로 행의 보조 버튼 스타일

### 데이터 요구사항

| 엔드포인트 | 데이터 |
|---|---|
| `GET /api/subscriptions/[id]` | 청약 상세 전체 객체 |
| `GET /api/subscriptions/[id]/supply` | 유형 및 순위별 공급 내역 |
| `GET /api/subscriptions/[id]/schedule` | 날짜별 타임라인 단계 |
| `GET /api/news?subscription=[id]&limit=5` | 관련 뉴스 기사 |

### 모바일 레이아웃

## 페이지 4: 뉴스 피드

- **라우트:** `/news`
- **목적:** 정책 변경, 시장 동향, 분석 기사를 제공. 사용자가 청약 관련 전반적인 흐름을 파악할 수 있도록 지원.

### 섹션

#### 4.1 카테고리 탭

- **위치:** 헤더 아래 고정
- **카테고리:**
  - 전체 — 기본값
  - 정책
  - 시장동향
  - 분석
  - 공지
- **스타일:** 모바일에서는 가로 스크롤, `primary` 밑줄 인디케이터

  - 썸네일 이미지 (16:9 비율, `placeholder.svg`를 대체 이미지로 사용)

  - 제목 (헤드라인, 2줄 제한)
  - 요약문 (본문, 3줄 제한)
  - 게시일 (상대 형식: "2일 전", "1주 전")
- **카드 스타일:** `white` 배경, `8px` 반경, 카드 간 32px 세로 간격
- 작성자명 또는 바이라인 표시 금지

#### 4.3 페이지네이션

- 청약 목록과 동일한 패턴
- 페이지당 15개 항목

### 데이터 요구사항

## 라우트 요약

| 라우트 | 페이지 | SSR | 인증 |
|---|---|---|---|
| `/` | 홈 대시보드 | Yes (ISR) | 공개 |
| `/listings` | 청약 목록 | Yes (SSR) | 공개 |
| `/listings/[id]` | 청약 상세 | Yes (SSG + ISR) | 공개 |
| `/news` | 뉴스 피드 | Yes (ISR) | 공개 |
| `/news/[id]` | 뉴스 기사 상세 | Yes (SSG + ISR) | 공개 |

**통합 검색**은 오버레이 컴포넌트 (라우트 없음) — `⌘K` 또는 검색 아이콘으로 실행.

- **ISR:** Incremental Static Regeneration (라우트별 재검증 주기 상이 — ARCHITECTURE.md 참조)
- **SSR:** Server-Side Rendering (실시간 데이터)
- **인증:** 모든 페이지는 공개. 콘텐츠 관리용 관리자 라우트는 이 문서의 범위에 포함되지 않음.

## 공통 데이터 패턴

### 상태 매핑

| 한국어 | 영문 키 | 색상 토큰 | 칩 라벨 |
|---|---|---|---|
| 접수중 | `accepting` | `success-50` bg / `success-700` text | 접수중 |
| 접수예정 | `upcoming` | `info-50` bg / `info-700` text | 접수예정 |
| 마감임박 | `closing_soon` | `warning-50` bg / `warning-700` text | 마감임박 |
| 발표대기 | `pending` | `warning-100` bg / `warning-700` text | 발표대기 |
| 발표일 | `result_today` | `warning-50` bg / `warning-700` text | 발표일 |
| 청약완료 | `closed` | `neutral-100` bg / `neutral-500` text | 청약완료 |

### 날짜 형식

- 절대 날짜: `YYYY.MM.DD` 형식 (예: `2026.04.15`)
- 상대 날짜 (뉴스): `N일 전`, `N주 전`, `N개월 전`
- 기간 표시: `2026.04.15 ~ 2026.04.17`

### 이미지 처리

- 모든 이미지는 `placeholder.svg`를 대체 이미지로 사용
- 화면 밖 이미지는 지연 로딩 적용
- WebP 형식 우선, JPEG 대체

### SEO 요구사항

- 모든 페이지는 `buildPageMetadata()`를 통해 고유한 `<title>`과 `<meta description>`을 갖는다
- 청약 상세: `{아파트명} 청약 일정 및 정보 | 청약닷컴`
- Coming-soon 상태의 `/trades`: `실거래가 | 청약닷컴` (브랜드 쿼리 포착을 위해 색인 유지)
- 모든 페이지에 Open Graph 및 Twitter Card 메타 태그 적용. 기본 OG 이미지는 `/og` edge 라우트가 렌더링하고, 페이지별로 `?title=`/`?subtitle=` 쿼리로 override
- 구조화 데이터 (JSON-LD): 전역 `Organization` + `WebSite`, 상세 페이지에 `RealEstateListing` + `BreadcrumbList` (SearchAction은 listings가 `?q=` 바인딩 후 재도입)
- GEO: `public/llms.txt`가 AI 검색 엔진을 위한 인용 친화 요약을 제공
